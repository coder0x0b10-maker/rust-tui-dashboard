## 1. 扩展目标
在现有的投资仪表板基础上，新增“投资雷达”功能，整合技术指标分析与视觉化预警，协助掌握市场趋势与进出场时机。

## 2. 技术指标整合 (Technical Indicators)
在 TUI 中计算并显示简单的技术指标。
- **MACD (平滑异同移动平均线)**：用于判断趋势与动能。
  - 计算方式：快线 (12日 EMA) - 慢线 (26日 EMA)，信号线为 9日 EMA。
  - TUI 显示：显示 MACD 柱状图的数值或符号（如 `▲` / `▼`）。
- **RSI (相对强弱指标)**：用于判断资产是否超买或超卖。
  - 计算方式：基于 14 日价格变动计算。
  - TUI 显示：数值 (0-100)，附带状态标签（超买/超卖）。

## 3. 视觉化预警系统 (Visual Alert System)
定义当指标触发特定条件时，TUI 界面的颜色变更或闪烁逻辑。
- **RSI 预警**：
  - `RSI < 30` (超卖)：文字显示为 **绿色 (Green)** 或加粗，提示可能的买入机会。
  - `RSI > 70` (超买)：文字显示为 **红色 (Red)** 或闪烁 (Blink)，提示可能的过热风险。
- **MACD 预警**：
  - 黄金交叉 (MACD 向上突破信号线)：整行或标签高亮为 **亮绿色 (Light Green)**。
  - 死亡交叉 (MACD 向下突破信号线)：整行或标签高亮为 **亮红色 (Light Red)**。

## 4. 异步数据流优化 (Async Data Flow Optimization)
优化现有的 Yahoo Finance API 抓取逻辑，支援多指标并行运算。
- **批次与并行请求**：使用 `tokio::spawn` 针对多个资产同时发起异步请求，并限制并发数（例如使用 `Semaphore`）避免触发 API 速率限制。
- **快取机制**：实作短时间的历史资料快取 (In-memory Cache)，减少对同一资产重复计算指标时的网路负担。
- **背景计算**：取得历史报价后，将 MACD 与 RSI 的运算放入背景的阻挡任务池 (`tokio::task::spawn_blocking`) 中执行，以免阻塞 TUI 的 UI 渲染执行绪。

## 5. 扩展 UI 设计 (Extended UI Design)
在原有的 `UI_DESIGN.md` 基础上，规划一个新的“Radar View”分页。
- **版面配置 (Radar View Layout)**：
  - **顶部导航列**：新增 `[3] Radar` 分页。
  - **左侧列表区**：追踪中的自选股清单（Ticker、当前价格、涨跌幅）。
  - **右侧指标面板**：
    - 上半部：详细报价与 RSI 指数仪表板（使用 TUI 的 Gauge 或 Sparks chart）。
    - 下半部：MACD 趋势图表与近期信号列表。
  - **底部状态列**：显示资料最后更新时间、API 请求状态。
