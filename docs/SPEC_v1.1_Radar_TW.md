## 1. 擴展目標
在現有的投資儀表板基礎上，新增「投資雷達」功能，整合技術指標分析與視覺化預警，協助掌握市場趨勢與進出場時機。

## 2. 技術指標整合 (Technical Indicators)
在 TUI 中計算並顯示簡單的技術指標。
- **MACD (平滑異同移動平均線)**：用於判斷趨勢與動能。
  - 計算方式：快線 (12日 EMA) - 慢線 (26日 EMA)，信號線為 9日 EMA。
  - TUI 顯示：顯示 MACD 柱狀圖的數值或符號（如 `▲` / `▼`）。
- **RSI (相對強弱指標)**：用於判斷資產是否超買或超賣。
  - 計算方式：基於 14 日價格變動計算。
  - TUI 顯示：數值 (0-100)，附帶狀態標籤（超買/超賣）。

## 3. 視覺化預警系統 (Visual Alert System)
定義當指標觸發特定條件時，TUI 介面的顏色變更或閃爍邏輯。
- **RSI 預警**：
  - `RSI < 30` (超賣)：文字顯示為 **綠色 (Green)** 或加粗，提示可能的買入機會。
  - `RSI > 70` (超買)：文字顯示為 **紅色 (Red)** 或閃爍 (Blink)，提示可能的過熱風險。
- **MACD 預警**：
  - 黃金交叉 (MACD 向上突破信號線)：整行或標籤高亮為 **亮綠色 (Light Green)**。
  - 死亡交叉 (MACD 向下突破信號線)：整行或標籤高亮為 **亮紅色 (Light Red)**。

## 4. 異步數據流優化 (Async Data Flow Optimization)
優化現有的 Yahoo Finance API 抓取邏輯，支援多指標並行運算。
- **批次與並行請求**：使用 `tokio::spawn` 針對多個資產同時發起非同步請求，並限制並發數（例如使用 `Semaphore`）避免觸發 API 速率限制。
- **快取機制**：實作短時間的歷史資料快取 (In-memory Cache)，減少對同一資產重複計算指標時的網路負擔。
- **背景計算**：取得歷史報價後，將 MACD 與 RSI 的運算放入背景的阻擋任務池 (`tokio::task::spawn_blocking`) 中執行，以免阻塞 TUI 的 UI 渲染執行緒。

## 5. 擴展 UI 設計 (Extended UI Design)
在原有的 `UI_DESIGN.md` 基礎上，規劃一個新的「Radar View」分頁。
- **版面配置 (Radar View Layout)**：
  - **頂部導航列**：新增 `[3] Radar` 分頁。
  - **左側列表區**：追蹤中的自選股清單（Ticker、當前價格、漲跌幅）。
  - **右側指標面板**：
    - 上半部：詳細報價與 RSI 指數儀表板（使用 TUI 的 Gauge 或 Sparks chart）。
    - 下半部：MACD 趨勢圖表與近期信號列表。
  - **底部狀態列**：顯示資料最後更新時間、API 請求狀態。
