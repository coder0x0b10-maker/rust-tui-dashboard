## 設計原則

1. **清晰**: 資訊一目瞭然，重點數據（總資產、報酬率）置頂。
2. **高效**: 全鍵盤操作，常用功能單鍵直達。
3. **美觀**: 運用 ASCII 字元與色彩區分資產類別。

## 鍵盤快捷鍵

| 按鍵 | 功能 |
|------|------|
| `P` | 開啟投資組合切換視窗 |
| `L` | 開啟交易歷史紀錄視窗 |
| `A` | 開啟新增交易視窗 |
| `R` | 開啟再平衡建議視窗 |
| `S` | 開啟系統設定視窗 |
| `H` | 顯示快捷鍵說明 (Help) |
| `Q` | 退出程式 |
| `Esc` | 返回上層選單或取消當前操作 |
| `Enter`| 確認輸入或選擇 |

---

## 視覺規範 (Color Palette)

- **數據變動**:
  - `Green`: 正報酬、買入 (Buy)
  - `Red`: 負報酬、賣出 (Sell)
  - `Yellow`: 警示、再平衡差異過大
- **UI 元件**:
  - `Cyan`: 標題、選取框架
  - `Magenta`: 指令提示、快捷鍵說明
  - `DarkGray`: 次要資訊、表格格線

---

## 錯誤與提示 (Status Bar Logic)

當發生錯誤或更新時，底部 Status Bar 將依據類型顯示：
- **INFO**: `[I] 股價已更新 (2026-02-24 13:45:01)` (藍底)
- **WARN**: `[W] Yahoo API 連線緩慢，使用快取數據` (黃底)
- **ERROR**: `[E] 資料庫寫入失敗: Disk Full` (紅底)
